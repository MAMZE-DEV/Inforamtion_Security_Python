# DES란?
오래된 금고로, 56비트의 열쇠를 사용.
16번동안 문서를 뒤섞고 변형하는 방식.
최근엔 강도가 약해서 해킹에 취약함

# AES란?
더 튼튼한 금고로, 128, 192, 256비트 길이의 열쇠를 사용
문서를 여러번 섞고, 치환하고 다시섞는 방식.
강도가 강하여 은행, 정부 등에서 많이 사용

## 구조 방식 차이
DES
- 데이터를 왼쪽과 오른쪽 두개로 나눔.
- 오른쪽 데이터를 복잡한 함수로 변형하여 왼쪽과 섞음.
- 이과정을 16번 반복
- 뒤섞이고 암호화된 데이터.

AES 
- 데이터를 16개의 작은 블록으로 나눔
- 조각들을 치환하고 섞고 행렬연산 등을 함.
- 이과정을 10~14번 반복
- 완전히 변형된 암호 데이터가 나옴.
